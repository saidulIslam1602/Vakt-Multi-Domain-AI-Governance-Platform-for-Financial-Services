# Frontend Inspiration — allegronordic.com

Source: https://www.allegronordic.com

---

## Overall aesthetic & feel

- Clean, minimal, premium B2B SaaS look. No clutter; lots of whitespace; confident typography.
- Nordic design sensibility: Simple, purposeful, no decoration for its own sake.
- Full-screen hero section with a single bold headline ("Building next generation of software business") and 2–3 CTAs (Learn Vision / View Portfolio / Join the Team). Classic SaaS hero.
- Scroll-driven narrative: Each section tells one story (About → How we work → Brands → Leadership → Contact). The "Scroll" call-to-action suggests smooth scroll transitions.

---

## Design patterns observed

| Pattern | What they do | What to take into case study UI |
|---------|-------------|----------------------------------|
| **Hero** | Large headline, 1-line sub, 2–3 CTA buttons | Use the same for presentation UI concept slide. |
| **Section labels** | Small uppercase label above main heading (e.g. "How We Work", "Leadership") | Good for visual hierarchy in slides and UI mockups. |
| **Brand cards** | Product cards with short name + 1-line description | Good pattern for document type or dashboard cards. |
| **3-column feature grid** | "Built Together / Speed to Value / Measurable Impact" | Classic B2B SaaS feature grid; shows they understand modern SaaS patterns. |
| **Leadership bios** | Photo + role + short 2–3 line summary, then expanded bio | Clean profile card pattern. |
| **Contact / footer** | Email + location, minimal, no noise | B2B contact: less form, more clarity. |
| **Careers page** | Role cards with department/location/type tags + "View Details" or "Coming soon" badge | Exactly the kind of list/detail/filter UI relevant to the document case study. |

---

## Colour & typography signals

- Mostly white/light background — premium, uncluttered.
- Dark, bold headings — probably dark grey or near-black; professional not stark.
- Accent colour likely blue or teal — typical for Nordic / B2B SaaS.
- Sans-serif fonts — modern, functional (likely Inter, Geist, or similar).
- Minimal icons or illustration — if any icons, they are simple line icons; no heavy illustration or stock photos.

---

## What this means for the case study UI concept

If including a frontend concept in the case presentation, match their style:

- A clean white dashboard, dark sidebar or top nav.
- Document list as cards (like their careers card list): filename, document type, extracted date, status badge (processing / ready / error).
- Detail/edit panel: metadata fields for extracted values (dates, parties, amounts, key terms) — all editable.
- Search bar at the top; full-text and semantic search results below.
- Chat/RAG panel on the right side or as a drawer — minimal, clean, message-style UI.
- Typography: Bold section headers, light body text, monospace for extracted values (dates, amounts) if needed.

**Components to sketch or mention in presentation:**
1. Document list view (cards with status and key metadata chips)
2. Document detail / edit view (extracted metadata fields, editable)
3. Search results view (keyword + semantic)
4. Chat / RAG panel (Q&A over document content)

## One-line summary

Premium, minimal, clean Nordic B2B SaaS design — a white dashboard with card lists, bold headings, lots of whitespace, and no visual noise would feel very at home as the frontend for the case study proposal.
